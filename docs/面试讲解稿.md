# 面试讲解稿

## 项目名称
RuoYi-Vue-Pro 后台管理系统接口与 UI 自动化测试项目

## 简历描述
基于 RuoYi-Vue-Pro 前后端分离后台管理系统，使用 Python + pytest + requests + Playwright 搭建综合自动化测试框架，覆盖登录认证、字典管理、用户管理、角色管理、菜单权限等核心模块，实现接口自动化、UI 自动化、接口造数、数据库校验、失败截图、Trace 回放、Allure 报告和 Jenkins 持续集成。

## 技术栈
Python、pytest、requests、Playwright、Page Object、YAML、pymysql、Allure、Jenkins、Git

## 6 大亮点
1. **接口和 UI 联动**：通过接口准备测试数据，UI 验证页面展示，接口清理数据。
2. **接口测试加入数据库校验**：对增删改类接口查库验证数据落库与状态变化。
3. **UI 自动化采用 Page Object 分层**：页面元素和操作封装进 pages 层，用例只写业务流程。
4. **权限场景设计**：接口建角色分配菜单、建用户绑角色，UI 登录验证菜单展示差异。
5. **失败定位能力**：UI 失败自动截图 + Playwright Trace 回放。
6. **持续集成**：Jenkins 自动拉代码、装依赖、跑测试、生成 Allure 报告。

## 9 问参考答案

### Q1 为什么选 RuoYi-Vue-Pro？
它是完整的前后端分离后台管理系统，包含登录、用户、角色、菜单权限、字典、部门等典型模块，接口丰富适合接口自动化，页面完整适合 Playwright UI 自动化。

### Q2 接口自动化框架怎么设计？
分 base、clients、testcases、common 四层。BaseApi 封装 get/post/put/delete、headers、token；clients 按业务模块封装接口方法；testcases 只写流程和断言；common 放配置、日志、YAML、数据库、断言工具。

### Q3 UI 自动化框架怎么设计？
Page Object 模型，分 base、pages、testcases、common 四层。BasePage 封装点击、输入、等待、截图、Toast 断言；pages 封装页面元素和动作；testcases 只写流程和断言。

### Q4 接口和 UI 怎么配合？
接口负责业务逻辑、数据校验、造数和清理；UI 负责验证真实操作和页面展示。如接口创建字典 → UI 查询验证 → 接口清理。

### Q5 怎么处理 token？
调登录接口拿 token，放在 session 级 fixture 复用，BaseApi 统一放进 Authorization 请求头。

### Q6 怎么处理测试数据？
查询类用系统已有稳定数据；增删改类用时间戳随机数据。结束优先接口清理，必要时数据库兜底。

### Q7 为什么要数据库校验？
接口返回成功不等于数据正确落库。对增删改类接口查库确认字段值和状态（含若依逻辑删除 deleted=1）。

### Q8 UI 用例不稳定怎么办？
从定位器（语义化）、等待机制（不用 sleep，用 expect）、数据隔离（随机+清理）、失败截图和 Trace 几方面处理。

### Q9 怎么做权限验证？
接口创建角色并分配指定菜单权限，创建用户绑角色，用 Playwright 登录该用户，验证它只能看到被授权菜单、看不到未授权菜单。
