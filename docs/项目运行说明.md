# 项目运行说明

> 本文档面向「拿到本项目代码后想要在本地跑起来」的读者，从被测系统启动、环境配置、各类测试运行到报告生成，全程手把手。
>
> 适合：面试前自查、同事接手、课程复现。

---

## 一、整体依赖关系

跑通本项目需要 4 个被测服务 + 1 个测试环境，缺一不可：

```
┌─────────────────────────────────────────────────────────┐
│  被测系统（RuoYi-Vue-Pro）                                │
│  ┌──────────┐  ┌──────────┐  ┌─────────┐  ┌──────────┐  │
│  │ 后端48080│  │ 前端80   │  │ MySQL   │  │ Redis    │  │
│  │Spring Boot│ │ Vue3+Vite│  │ 3306    │  │ 6379     │  │
│  └──────────┘  └──────────┘  └─────────┘  └──────────┘  │
└─────────────────────────────────────────────────────────┘
              ▲
              │ HTTP / 浏览器 / JDBC
┌─────────────┴───────────────────────────────────────────┐
│  测试项目（ruoyi-auto-test）                              │
│  Python 3.11 + pytest + Playwright + Allure             │
└─────────────────────────────────────────────────────────┘
```

## 二、启动被测系统

### 1. 启动 MySQL

```bash
# Docker 方式（推荐）
docker run -d --name ruoyi-mysql -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  mysql:8.0

# 或使用本机已安装的 MySQL，确保 3306 可连、root 密码 123456
```

导入初始化 SQL（来自 RuoYi-Vue-Pro 源码 `sql/` 目录）：

```bash
mysql -uroot -p123456 -e "CREATE DATABASE IF NOT EXISTS \`ruoyi-vue-pro\` DEFAULT CHARSET utf8mb4;"
mysql -uroot -p123456 ruoyi-vue-pro < /path/to/ruoyi-vue-pro/sql/mysql/ruoyi-vue-pro.sql
```

### 2. 启动 Redis

```bash
docker run -d --name ruoyi-redis -p 6379:6379 redis:7
# 或本机已安装的 Redis
```

### 3. 启动后端（端口 48080）

后端源码位于 `E:/ruoyi`（RuoYi-Vue-Pro / yudao-server）：

```bash
cd E:/ruoyi/yudao-server
# 修改 application-local.yaml 中数据库连接（库名 ruoyi-vue-pro、密码 123456）
mvn spring-boot:run
# 或在 IDEA 中运行 YudaoServerApplication 主类
```

确认后端启动成功：

```bash
curl http://localhost:48080/admin-api/system/auth/login \
  -X POST -H "Content-Type: application/json" -H "tenant-id: 1" \
  -d '{"username":"admin","password":"admin123"}'
# 期望返回 200 且包含 accessToken
```

### 4. 启动前端（端口 80）

前端源码位于 `E:/ruoyi/yudao-ui/yudao-admin-vue3`：

```bash
cd E:/ruoyi/yudao-ui/yudao-admin-vue3
pnpm install   # 或 npm install
# 修改 .env.local 中 VITE_BASE_URL=http://localhost:48080
pnpm dev       # 默认 80 端口
```

确认前端启动成功：

```bash
curl -o /dev/null -w "%{http_code}" http://localhost:80
# 期望 200
```

### 5. 确认 MySQL / Redis 正常

```bash
# MySQL
mysql -uroot -p123456 -e "SELECT 1 FROM ruoyi-vue-pro.system_users LIMIT 1"
# 期望返回 admin 用户记录

# Redis
redis-cli ping
# 期望 PONG
```

> **关键前置条件**：测试登录接口依赖「图形验证码关闭」。RuoYi-Vue-Pro 默认本地环境已关闭验证码，若你改过 `application-local.yaml` 的 `yudao.captcha.enable`，请确保其为 `false`。

## 三、配置测试项目

### 1. 安装 Python 依赖

```bash
cd E:/ruoyi/test/ruoyi-auto-test
python -m venv .venv
.venv/Scripts/activate          # Windows Git Bash
# source .venv/bin/activate     # Linux/Mac
pip install -r requirements.txt
```

### 2. 安装 Playwright 浏览器

```bash
playwright install chromium
# 如需系统依赖（Linux）：playwright install-deps chromium
```

### 3. 配置 `data/env.yaml`

复制示例配置并按本地实际修改：

```bash
cp data/env.example.yaml data/env.yaml
```

`data/env.yaml` 内容（本地开发用）：

```yaml
env:
  base_url: "http://localhost:48080"
  web_url: "http://localhost:80"
  tenant_id: "1"
  tenant_name: "芋道源码"

admin:
  username: "admin"
  password: "admin123"

db:
  host: "127.0.0.1"
  port: 3306
  user: "root"
  password: "123456"
  database: "ruoyi-vue-pro"
```

> `data/env.yaml` 已在 `.gitignore` 中，**不会被提交**，可放心填写真实密码。
>
> 也可改用环境变量（优先级高于 yaml），见 README「配置方式」一节，适合 CI 环境。

## 四、运行测试

所有命令在项目根目录 `E:/ruoyi/test/ruoyi-auto-test` 下执行。

### 1. 只跑 API 接口测试（103 条）

```bash
pytest api_auto/testcases -q --alluredir=reports/allure-results
```

### 2. 只跑 UI 自动化测试（70 条）

```bash
pytest ui_auto/testcases -q --alluredir=reports/allure-results
```

> UI 测试默认 headless。调试时加 `--headed` 看浏览器，加 `--slowmo 500` 放慢操作。

### 3. 只跑接口联动 / DB 校验测试（20 条）

```bash
pytest integration -q --alluredir=reports/allure-results
```

### 4. 全量运行（205 条）

```bash
pytest -q --alluredir=reports/allure-results
```

### 5. 按 marker 运行

```bash
pytest -m smoke          # 只跑冒烟用例
pytest -m api            # 只跑 api 标记
pytest -m "db"           # 只跑含数据库校验的用例
pytest -m "flow"         # 只跑联动用例
```

### 6. 运行单个用例

```bash
pytest api_auto/testcases/test_auth_api.py::TestAuthApi::test_login_success -v
```

项目按串行方式运行，便于初学阶段观察日志、数据清理和失败位置。

## 五、生成与查看 Allure 报告

### 1. 生成静态报告

```bash
allure generate reports/allure-results -o reports/allure-report --clean
```

### 2. 在线预览（自动刷新）

```bash
allure serve reports/allure-results
# 自动打开浏览器 http://localhost:<随机端口>
```

### 3. 直接打开静态报告

```bash
# Windows
start reports/allure-report/index.html
# 或用脚本（见下文）：scripts/open_allure.bat
```

报告内容包含：
- **Overview**：总览，通过/失败/跳过数、耗时分布
- **Suites**：按测试套件（文件/类）分组查看
- **Categories**：失败分类（本项目全绿时为空）
- **Timeline**：用例执行时间线
- 每条用例的步骤、请求/响应、数据库记录、截图、日志等附件

## 六、一键运行脚本（Windows）

项目提供 `scripts/` 目录下的批处理脚本，双击或在命令行执行：

| 脚本 | 作用 |
|------|------|
| `scripts/run_api.bat` | 只跑 API 测试并生成 Allure 结果 |
| `scripts/run_ui.bat` | 只跑 UI 测试并生成 Allure 结果 |
| `scripts/run_all.bat` | 全量运行 |
| `scripts/run_collect.bat` | 只收集用例不执行（验证用例可加载） |
| `scripts/open_allure.bat` | 生成并打开 Allure 报告 |

## 七、常见报错与解决办法

### 1. `ConnectionRefusedError: 48080 连接失败`

**原因**：后端未启动或端口不对。

**解决**：
```bash
curl http://localhost:48080/admin-api/system/auth/login -X POST ...
# 若失败，检查后端进程、application-local.yaml 端口配置
```

### 2. `playwright._impl._api_types.Error: Executable doesn't exist`

**原因**：未安装 Playwright 浏览器。

**解决**：
```bash
playwright install chromium
```

### 3. `pymysql.err.OperationalError: Access denied for user 'root'`

**原因**：`data/env.yaml` 中数据库密码与实际不符。

**解决**：核对 `db.password`，确保与 MySQL `root` 密码一致。

### 4. `AssertionError: code != 0`（登录用例失败）

**原因**：常见为验证码未关闭，或账号密码错误。

**解决**：
- 确认后端 `application-local.yaml` 中 `yudao.captcha.enable: false`
- 确认 `admin/admin123` 可登录（前端手动试一次）

### 5. UI 用例报 `TimeoutError: page.goto: net::ERR_CONNECTION_REFUSED`

**原因**：前端 80 端口未启动。

**解决**：启动前端 `pnpm dev`，确认 `http://localhost:80` 可访问。

### 6. `ModuleNotFoundError: No module named 'xxx'`

**原因**：未激活 venv 或未装依赖。

**解决**：
```bash
.venv/Scripts/activate
pip install -r requirements.txt
```

### 7. Allure 报告打开是空白页

**原因**：直接用 `file://` 打开 `index.html` 时，浏览器安全策略阻止加载 data 文件。

**解决**：用 `allure serve` 启动本地服务，或用 `allure open reports/allure-report`。

### 8. UI 测试在 CI 服务器上跑不了

**原因**：CI 环境无图形界面或缺系统依赖。

**解决**：保持 headless（默认即 headless），Linux 上执行 `playwright install-deps chromium` 安装系统库。

### 9. 测试数据残留导致用例失败

**原因**：上次运行异常中断，测试前缀数据未清理。

**解决**：项目已实现 `TEST_RUN_PREFIX` 兜底清理，但极端情况下可手动清理：
```sql
DELETE FROM system_dict_type WHERE type LIKE 'auto_%';
DELETE FROM system_role WHERE name LIKE 'auto_%';
-- 其他表同理
```

### 10. `pytest: error: unrecognized arguments: --alluredir`

**原因**：未安装 `allure-pytest` 插件。

**解决**：
```bash
pip install allure-pytest
```

## 八、验证环境是否就绪（一键自检）

依次执行以下命令，全部成功即可开始测试：

```bash
# 1. 后端
curl -sf -o /dev/null http://localhost:48080/admin-api/system/auth/login -X POST \
  -H "Content-Type: application/json" -H "tenant-id: 1" \
  -d '{"username":"admin","password":"admin123"}' && echo "后端 OK"

# 2. 前端
curl -sf -o /dev/null http://localhost:80 && echo "前端 OK"

# 3. MySQL
mysql -uroot -p123456 -e "SELECT 1" && echo "MySQL OK"

# 4. Redis
redis-cli ping && echo "Redis OK"

# 5. Python 依赖
.venv/Scripts/python.exe -c "import pytest,playwright,requests,pymysql,yaml; print('依赖 OK')"

# 6. Playwright 浏览器
.venv/Scripts/python.exe -m pytest --collect-only -q | tail -1
# 期望输出 "205 tests collected"
```

全绿后执行 `pytest -q` 即可运行全量测试。

## 九、目录产物说明

运行后会产生以下目录（均已在 `.gitignore` 中）：

| 目录 | 内容 |
|------|------|
| `reports/allure-results/` | Allure 原始结果数据（JSON） |
| `reports/allure-report/` | Allure 静态 HTML 报告 |
| `screenshots/` | 失败用例自动截图 |
| `traces/` | 失败用例 Playwright Trace 包 |
| `logs/` | 运行日志 |
| `__pycache__/` | Python 字节码缓存 |

> 成功用例不落盘截图和 Trace，避免产物膨胀；只有失败用例才生成，便于定位。
