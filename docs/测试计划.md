# 测试计划

## 1. 测试目标
验证 RuoYi-Vue-Pro 后台管理系统核心模块的接口、UI、接口+UI联动流程是否符合预期，并形成可复用自动化测试框架。

## 2. 测试范围
- 登录认证
- 字典管理
- 部门管理
- 岗位管理
- 用户管理
- 角色管理
- 菜单权限
- 接口+UI联动
- 数据库校验
- JMeter 性能测试场景设计

## 3. 非测试范围
- 支付、工作流、商城订单、复杂多租户、大屏报表、安全深度测试。

## 4. 测试环境
- 后端：http://localhost:48080
- 前端：http://localhost:80
- 数据库：MySQL ruoyi-vue-pro
- Redis：本地或测试环境
- 验证码：本地自动化测试环境建议关闭 `yudao.captcha.enable=false`

## 5. 进入准则
- MySQL、Redis、后端、前端可启动；
- 管理员账号可登录；
- 接口文档可访问；
- 数据库账号可查询测试表。

## 6. 退出准则
- 核心接口用例、UI用例、联动用例执行完成；
- 失败用例有明确原因；
- 自动化创建的测试数据已清理；
- 生成 Allure 报告和测试总结。

## 7. 风险与应对
| 风险 | 应对 |
|---|---|
| 验证码导致接口登录失败 | 测试环境关闭验证码 |
| UI定位随前端版本变化 | 优先语义化定位，失败后用 Trace 回放 |
| 测试数据残留 | auto 前缀 + 接口清理 + DB兜底清理 |
| token 被 logout 污染 | logout 使用独立 token |
| Allure 泄露敏感信息 | 日志和附件统一脱敏 |
