## 设计要点说明

1. **编号规则**：`模块_类型_序号`，如 `AUTH_API_001`、`DICT_UI_006`、`USER_FLOW_003`，便于在 Allure 报告中按编号检索。
2. **API+DB 标记**：API 用例中带 `@pytest.mark.db` 的 7 条，验证数据真实落库（`system_*` 表的 `deleted` 逻辑删除字段）。
3. **UI 造数策略**：UI 用例中 47 条采用「API 造数 + UI 验证」——复杂表单数据由 API 预置，UI 层聚焦页面打开、查询、展示、必填校验、删除确认等真实页面行为，避免 UI 层陷入冗长表单填写。
4. **联动闭环**：Integration 用例体现「接口造数→接口验证→DB 校验→清理」完整闭环，验证三层一致性（接口返回、页面数据、数据库记录）。
5. **清理保障**：每条用例 `finally` 中按本轮 `TEST_RUN_PREFIX` 兜底清理，保证用例幂等可重复执行。
