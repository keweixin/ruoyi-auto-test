# 接口分析（RuoYi v3.9.2 原版）

> 本文档记录当前测试项目实际使用的 RuoYi v3.9.2 原版接口，不再使用 RuoYi-Vue-Pro / yudao 的 `/admin-api` 体系。

## 通用约定

| 项 | 值 |
|----|----|
| 后端地址 | `http://localhost:8080` |
| 全局前缀 | 无，接口直接从 `/login`、`/system/...` 开始 |
| 鉴权头 | `Authorization: Bearer <token>` |
| 成功业务码 | `code == 200` |
| token 字段 | 登录返回 `token` |
| 分页字段 | `pageNum` / `pageSize` |
| 分页结果 | `rows` / `total` |
| 非分页列表 | 多数返回 `data` |
| 验证码 | 数据库 `sys_config.sys.account.captchaEnabled=false` |

## 登录认证

| 方法 | 路径 | 说明 | 返回 |
|---|---|---|---|
| POST | `/login` | 登录 | `token` |
| POST | `/logout` | 退出 | `code=200` |
| GET | `/getInfo` | 当前用户信息 | `user/roles/permissions` |

## 字典类型 `sys_dict_type`

| 方法 | 路径 | 入参 |
|---|---|---|
| GET | `/system/dict/type/list` | `pageNum,pageSize,dictName?,dictType?` |
| POST | `/system/dict/type` | `dictName,dictType,status,remark?` |
| PUT | `/system/dict/type` | `dictId,dictName,dictType,status` |
| DELETE | `/system/dict/type/{dictId}` | 路径参数 |
| GET | `/system/dict/type/{dictId}` | 路径参数 |

## 字典数据 `sys_dict_data`

| 方法 | 路径 | 入参 |
|---|---|---|
| GET | `/system/dict/data/list` | `dictType,dictLabel?,status?` |
| POST | `/system/dict/data` | `dictSort,dictLabel,dictValue,dictType,status` |
| PUT | `/system/dict/data` | `dictCode,dictSort,dictLabel,dictValue,dictType,status` |
| DELETE | `/system/dict/data/{dictCode}` | 路径参数 |

## 部门 `sys_dept`

| 方法 | 路径 | 入参 |
|---|---|---|
| GET | `/system/dept/list` | `deptName?,status?`，返回 `data` |
| POST | `/system/dept` | `parentId,deptName,orderNum,status` |
| PUT | `/system/dept` | `deptId,parentId,deptName,orderNum,status` |
| DELETE | `/system/dept/{deptId}` | 路径参数，逻辑删除 `del_flag=2` |
| GET | `/system/dept/{deptId}` | 详情 |

## 岗位 `sys_post`

| 方法 | 路径 | 入参 |
|---|---|---|
| GET | `/system/post/list` | `pageNum,pageSize,postName?,postCode?` |
| POST | `/system/post` | `postCode,postName,postSort,status` |
| PUT | `/system/post` | `postId,postCode,postName,postSort,status` |
| DELETE | `/system/post/{postId}` | 路径参数 |

## 用户 `sys_user`

| 方法 | 路径 | 入参 |
|---|---|---|
| GET | `/system/user/list` | `pageNum,pageSize,userName?,phonenumber?` |
| POST | `/system/user` | `userName,nickName,password,phonenumber,deptId,status` |
| PUT | `/system/user` | `userId,userName,nickName,phonenumber,deptId` |
| PUT | `/system/user/changeStatus` | `userId,status` |
| PUT | `/system/user/resetPwd` | `userId,password` |
| DELETE | `/system/user/{userId}` | 路径参数，逻辑删除 |

## 角色 `sys_role`

| 方法 | 路径 | 入参 |
|---|---|---|
| GET | `/system/role/list` | `pageNum,pageSize,roleName?` |
| POST | `/system/role` | `roleName,roleKey,roleSort,status,menuIds` |
| PUT | `/system/role` | `roleId,roleName,roleKey,roleSort,status,menuIds` |
| DELETE | `/system/role/{roleId}` | 路径参数 |
| GET | `/system/role/{roleId}` | 详情 |

## 菜单与权限

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/system/menu/list` | 菜单列表，返回 `data` |
| GET | `/system/menu/roleMenuTreeselect/{roleId}` | 查询角色菜单树和 checkedKeys |
| POST/PUT | `/system/role` | 通过 `menuIds` 创建/更新角色菜单关系 |

## 关键数据库表

| 表名 | 说明 | 删除方式 |
|---|---|---|
| `sys_dict_type` | 字典类型 | 物理删除 |
| `sys_dict_data` | 字典数据 | 物理删除 |
| `sys_dept` | 部门 | `del_flag=2` |
| `sys_post` | 岗位 | 物理删除 |
| `sys_user` | 用户 | `del_flag=2` |
| `sys_role` | 角色 | `del_flag=2` |
| `sys_menu` | 菜单 | 物理删除 |
| `sys_role_menu` | 角色菜单关系 | 关系表 |
| `sys_user_role` | 用户角色关系 | 关系表 |
