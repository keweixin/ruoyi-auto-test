# 测试总结

## 1. 测试范围
本项目设计/实现 **205 个测试实例**，覆盖接口、UI、接口联动与数据库校验：
- 接口：登录、字典、部门、岗位、用户、角色、菜单、操作日志、登录日志、个人中心、通知公告（10 模块，103 条）
- UI：登录、字典、部门、岗位、用户、角色、菜单（7 模块，70 条，含 9 条 REAL 真实 UI 操作用例）
- 联动：字典、用户、角色权限、部门岗位、接口操作后 DB 校验（20 条）
- 框架单测：BaseApi HTTP 客户端 + common 工具方法（12 条）

> 当前 README、用例清单和本总结均以最近一次 pytest/Allure 执行结果为准。

## 2. 执行环境
- 本地 Windows
- 后端：`localhost:48080`，管理端接口 `/admin-api`
- 前端：`localhost:80`，Vue3 + Element Plus + Vite
- MySQL：Docker/本机 3306，库 `ruoyi-vue-pro`
- Redis：Docker/本机 6379
- 鉴权：`tenant-id: 1` + `Authorization: Bearer <data.accessToken>`
- pytest + requests + Playwright + Allure

## 4. 已完成改进
- 将测试协议从 RuoYi 原版重新适配为 RuoYi-Vue-Pro / yudao
- 登录改为 `/admin-api/system/auth/login`，token 改为 `data.accessToken`
- 成功断言改为 `code == 0`
- BaseApi 统一拼 `/admin-api`，统一携带 `tenant-id` 和 Authorization
- 数据库表统一改为 `system_*`，逻辑删除使用 `deleted`
- UI 登录适配 Vue3 登录页租户/用户名/密码输入框
- TokenManager 支持过期前刷新、业务码 401 自动恢复和限次重试
- LoginPage + AuthStateManager 统一复用 UI 登录态，状态文件损坏或失效时自动重建
- UI 用例优先使用 API 造数 + UI 查询验证，降低 Element Plus 表单不稳定性
- 共享 fixture 上移根 conftest，联动测试可访问
- logout 使用独立 token，避免污染 session token
- 本轮 API/UI 签发 Token 统一登记并在 session 结束时注销
- 兜底清理覆盖用户角色、角色菜单、用户岗位和测试用户 OAuth2 Token 关系
- 日志与 Allure 脱敏
- UI 删除增加 auto 前缀保护与唯一行检查
- Trace 仅失败保存，成功用例不落盘

## 5. 风险
- UI 定位依赖前端版本，需根据本地页面微调
- JMeter 性能测试结果受本机硬件、数据库和 Redis 配置影响
- JMeter 当前只提供 `.jmx` 和操作文档，未产出真实 `.jtl` 或 HTML 性能报告
- 全量通过数量必须以真实执行报告为准
