# 初学者代码导读

## 先记住一条主线

```text
测试用例 → Client → BaseApi → 后端接口 → 断言
```

以“新增用户”为例：

1. `test_user_api.py` 调用 `data/builders.py`中的 `valid_user_data()`准备请求字典。
2. pytest 把 `user_client` fixture 传给测试方法。
3. `user_client.create(data)`拼接用户接口路径。
4. `BaseApi`统一加入 `tenant-id`和 Token，然后用 requests 发请求。
5. 测试用例使用 `assert_response_ok()`或 `assert_api_ok()`断言。
6. 用例使用 `try/finally`删除自己创建的数据；全局清理只是兜底。

`BaseApi`只对 GET、HEAD、OPTIONS 自动重试，避免 POST 新增数据被重复提交。

## 五层结构

| 层 | 主要目录 | 只负责什么 |
|---|---|---|
| 配置层 | `data/env.yaml`、`common/config.py` | 环境地址、账号、数据库配置 |
| 工具层 | `common/` | 日志、断言、数据库、YAML、随机数据 |
| 接口层 | `api_auto/base`、`api_auto/clients` | 发送请求、封装接口地址 |
| 数据层 | `data/*.yaml`、`data/builders.py` | 参数化数据和少量合法请求体 |
| 测试层 | `api_auto/testcases`、`ui_auto/testcases`、`integration` | 测试步骤、断言、清理 |

## 初学阶段暂时跳过

以下代码是框架稳定性能力，不影响你先理解测试主线：

- `common/token_registry.py`：测试结束后注销本轮 Token。
- `ui_auto/auth/auth_state_manager.py`：复用 UI 登录状态。
- `pytest_runtest_makereport`：失败时截图和保存 Trace。
- `common/security_utils.py`：日志中的密码、Token 脱敏。

掌握主线后再逐个学习，不需要一次全部背下来。

## 编写新接口用例的固定模板

```python
def test_create_user(user_client):
    data = valid_user_data()
    user_id = None

    try:
        response = user_client.create(data)
        body = assert_response_ok(response, "创建用户")
        user_id = body["data"]

        detail = user_client.get(user_id).json()
        assert detail["data"]["username"] == data["username"]
    finally:
        if user_id:
            user_client.delete(user_id)
```

这个模板有四步：准备数据、发送请求、断言、清理。普通 CRUD 用例优先保持这种显式写法。

## conftest.py 怎么看

`conftest.py`不是测试用例，它给测试提供 fixture：

```python
def test_page_user(user_client):
    ...
```

参数名 `user_client`与 fixture 名相同，pytest 会自动创建并传入，不需要手动 import。

项目根目录的 `conftest.py`保存真正实现；`api_auto/conftest.py`和
`ui_auto/conftest.py`只说明 fixture 已上移，避免重复定义。
