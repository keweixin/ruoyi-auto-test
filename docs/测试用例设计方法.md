# 测试用例设计方法

## 1. 等价类划分
把输入分为有效等价类和无效等价类。例如登录：
- 有效：正确用户名 + 正确密码
- 无效：错误密码、空用户名、空密码、不存在用户

## 2. 边界值分析
关注长度、范围、分页边界等。例如：
- 用户名 4 位/30 位边界
- 密码 4 位/16 位边界
- pageNo=1、pageSize=1/10/100

## 3. 错误推测
根据经验补充容易出错的场景：
- 重复字典类型
- 重复岗位编码
- 删除被引用数据
- 未携带 token 访问受保护接口

## 4. 场景法
按照真实业务流程设计：
- 接口创建字典 → UI 查询 → 接口清理
- 接口创建用户 → UI 登录验证
- 接口创建角色分配菜单 → UI 验证菜单差异

## 5. 权限矩阵
| 用户类型 | 授权菜单 | 未授权菜单 | 预期 |
|---|---|---|---|
| 管理员 | 全部 | 无 | 可见全部菜单 |
| 测试角色用户 | 字典管理 | 用户管理/角色管理 | 只能看到授权菜单 |

## 6. 接口/UI/联动分工
- 接口：业务逻辑、异常、权限、数据库、造数、清理
- UI：真实操作、页面展示、Toast、表格、弹窗、菜单
- 联动：接口造数 + UI验证 + 接口清理 + DB校验
