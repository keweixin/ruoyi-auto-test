# 项目理解

## 架构图

```text
浏览器 / Playwright
   │  ① 访问页面 http://localhost:80
   ▼
前端 Vue3 + Element Plus + Vite（yudao-admin-vue3）
   │  ② 通过 /admin-api 调用后端接口
   ▼
后端 Spring Boot 2.7（端口 48080）
   │  ③ JDBC 读写数据库，Redis 支撑缓存/验证码/token 等能力
   ▼
MySQL（Docker/本机 3306，库 ruoyi-vue-pro）
Redis（Docker/本机 6379）
```

## 被测系统

当前项目实际适配的是 **RuoYi-Vue-Pro / yudao** 管理后台。

关键约定：

| 项 | RuoYi-Vue-Pro / yudao |
|---|---|
| 后端地址 | `http://localhost:48080` |
| 管理端前缀 | `/admin-api` |
| 前端地址 | `http://localhost:80` |
| 登录接口 | `POST /admin-api/system/auth/login` |
| 鉴权头 | `tenant-id: 1` + `Authorization: Bearer <accessToken>` |
| 成功业务码 | `code == 0` |
| token 字段 | `data.accessToken` |
| 数据库 | `ruoyi-vue-pro` |
| 用户表 | `system_users` |
| 角色表 | `system_role` |
| 字典表 | `system_dict_type` / `system_dict_data` |

## 接口测试 vs UI 测试

- **接口测试**：直接请求 `/admin-api` 后端接口，验证业务逻辑、异常、鉴权和数据库结果。
- **UI 测试**：使用 Playwright 操作 Vue3 + Element Plus 页面，验证真实用户操作、表格、弹窗和菜单。
- **接口联动/DB 校验**：接口造数 → 接口查询验证 → 数据库 `system_*` 表校验 → 接口清理。

## 为什么选择 RuoYi-Vue-Pro

RuoYi-Vue-Pro 是典型后台管理系统，包含登录、用户、角色、菜单、字典、部门、岗位等模块，适合展示接口自动化、UI 自动化、权限验证、数据库校验、Allure 报告、Jenkins 流水线和 JMeter 性能脚本设计能力。
