# 项目理解

## 架构图

```text
浏览器 / Playwright
   │  ① 访问页面 http://localhost:8081
   ▼
前端 Vue2 + element-ui（RuoYi 原版 ruoyi-ui）
   │  ② 通过 /dev-api 代理到后端接口
   ▼
后端 Spring Boot（Docker: ruoyi-backend，端口 8080）
   │  ③ JDBC 读写数据库
   ▼
MySQL（Docker: ruoyi-mysql，端口 13306，库 ry-vue）
Redis（Docker: ruoyi-redis）
```

## 被测系统

当前项目实际适配的是 **RuoYi v3.9.2 原版**，不是 RuoYi-Vue-Pro / 芋道版本。

关键差异：

| 项 | RuoYi v3.9.2 原版 |
|---|---|
| 后端地址 | `http://localhost:8080` |
| 前端地址 | `http://localhost:8081` |
| 登录接口 | `POST /login` |
| 鉴权头 | `Authorization: Bearer <token>` |
| 成功业务码 | `code == 200` |
| token 字段 | `token` |
| 数据库 | `ry-vue` |
| 用户表 | `sys_user` |
| 角色表 | `sys_role` |
| 字典表 | `sys_dict_type` / `sys_dict_data` |

## 接口测试 vs UI 测试

- **接口测试**：直接请求后端接口，验证业务逻辑、异常、鉴权和数据库结果。
- **UI 测试**：使用 Playwright 操作前端页面，验证真实用户操作、表格、弹窗和菜单。
- **接口联动/DB 校验**：接口造数 → 接口查询验证 → 数据库校验 → 接口清理。

## 为什么选择若依

RuoYi v3.9.2 是典型后台管理系统，包含登录、用户、角色、菜单、字典、部门、岗位等模块，适合展示接口自动化、UI 自动化、权限验证、数据库校验、Allure 报告和 Jenkins 流水线设计能力。
