# 项目理解

## 架构图

```
浏览器(用户)
   │  ① 点击页面
   ▼
前端 Vue (yudao-ui)         —— 渲染页面、收集输入，端口 80
   │  ② HTTP 调后端接口 /admin-api/...
   ▼
后端 Spring Boot (yudao-server + yudao-module-*)  —— 业务逻辑、鉴权，端口 48080
   │  ③ SQL
   ▼
数据库 MySQL (ruoyi-vue-pro)  —— 存数据
```

## 接口测试 vs UI 测试
- **接口测试**：绕过前端，直接打第 ②③ 步，验证业务逻辑/数据/异常/权限/数据库落库。快、稳。
- **UI 测试**：走完整 ①②③，验证真实操作/页面展示/弹窗/Toast/菜单。慢、像真人。
- **联动**：接口造数 → UI 验证展示 → 接口清理。

## 为什么选若依
前后端分离、接口丰富、页面完整、有完整权限体系，既能做接口自动化，又能做 Playwright UI 自动化，还能做接口+UI联动。
第一阶段聚焦 `yudao-module-system`：用户/角色/菜单/字典/部门/岗位。
